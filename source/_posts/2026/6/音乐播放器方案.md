「兴趣爱好」页面的「音乐」Tab 采用了一套**零第三方依赖、纯原生 HTML5 Audio** 的自研播放器方案。本文梳理其数据流、渲染结构与播放逻辑，作为后续维护与重构的参考。

## 整体架构

播放器由三层组成，职责清晰分离：

```
music.yml (数据)  ──►  hobbies.ejs (渲染卡片 + 硬编码映射)  ──►  原生 Audio API (播放)
                                              │
                                              ▼
                                  GitHub Raw 仓库 (mp3 源)
```

| 层 | 文件 | 职责 |
| --- | --- | --- |
| 数据层 | [source/_data/music.yml](source/_data/music.yml) | 歌曲元数据（标题、歌手、日期、注释、跳转链接） |
| 渲染层 | [themes/Zebin-Theme/layout/hobbies.ejs](themes/Zebin-Theme/layout/hobbies.ejs) | 渲染音乐卡片 + 内嵌播放器脚本 |
| 音频源 | `ZebinGao/MyFavouriteMusic` GitHub 仓库 | 存放实际 mp3 文件 |
| 样式 | [themes/Zebin-Theme/source/css/style.css](themes/Zebin-Theme/source/css/style.css) | 卡片播放按钮、浮动播放器、进度条 |

## 数据层：music.yml

每首歌是一条结构化记录，**不含音频文件名**：

```yaml
- title: "吻别"
  artist: "张学友"
  date: "1993-03-05"
  note: "我和你吻别在无人的街，让风痴笑我不能拒绝。"
  url: "/wiki/music/#吻别"   # 卡片点击跳转到 wiki 详情页
```

卡片本身是可点击的 `<a>`（带 `url`）或 `<div>`（无 `url`），整张卡片承载跳转逻辑，**播放按钮是叠在卡片上的独立元素**。

## 渲染层：音乐卡片

[hobbies.ejs:89-116](themes/Zebin-Theme/layout/hobbies.ejs#L89-L116) 遍历 `site.data.music`，每张卡片渲染出标题、歌手/年份、注释，并附一个播放按钮：

```html
<a class="hobby-card music-card" href="..." data-title="吻别" data-artist="张学友">
    <button class="music-play-btn" data-title="吻别" data-artist="张学友">▶</button>
    <div class="hobby-card-title">吻别</div>
    <div class="hobby-card-meta">张学友 · 1993</div>
</a>
```

> 关键：按钮通过 `data-title` 携带歌名，作为后续查找音频文件的索引键。

## 核心设计：硬编码映射表 `mf`

这是整套方案最关键、也最值得记录的特点——**「歌名 → mp3 文件名」的映射被硬编码在模板的 `<script>` 里**，而非来自数据文件：

```javascript
var base = 'https://raw.githubusercontent.com/ZebinGao/MyFavouriteMusic/main/%E9%9F%B3%E4%B9%90/';
var mf = {
    '小夜曲': 'FranzSchubert-小夜曲.mp3',
    '吻别': '张学友-吻别.mp3',
    '蓝莲花': '许巍_蓝莲花.mp3',
    // ... 共 100+ 首
};
```

完整映射见 [hobbies.ejs:204-316](themes/Zebin-Theme/layout/hobbies.ejs#L204-L316)。

**渲染时按映射是否存在来决定是否显示播放按钮**——只有能播的歌才会露出 ▶：

```javascript
playBtns.forEach(function(b) {
    if (mf[b.dataset.title]) b.classList.add('has-audio');  // CSS 仅对 .has-audio 显示
});
```

## 播放逻辑

### 1. 单例 Audio 元素

全局只创建一个 `new Audio()`，所有歌曲复用它（[hobbies.ejs:318](themes/Zebin-Theme/layout/hobbies.ejs#L318)）。切换歌曲时只改 `src`，不新建对象。

### 2. 点击播放

```javascript
function playSong(title, artist) {
    var file = mf[title];
    if (!file) return;
    // 再次点击同一首歌 → 暂停
    if (curTitle === title && !audio.paused) { audio.pause(); updateUI(false); return; }
    // 换歌 → 拼接 URL（encodeURIComponent 处理中文/特殊字符文件名）
    if (curTitle !== title) { audio.src = base + encodeURIComponent(file); curTitle = title; }
    audio.play();
    // 弹出底部浮动播放器，更新标题/歌手，高亮当前卡片
}
```

### 3. 浮动播放器

[hobbies.ejs:120-134](themes/Zebin-Theme/layout/hobbies.ejs#L120-L134) 是一个固定在底部的播放条，首次播放后通过 `.active` 类 `display:block` 滑入（`slideUp` 动画）。包含：

- **播放/暂停按钮**：图标在 `▶` / `❚❚` 间切换
- **标题 + 歌手**：单行省略
- **时间显示**：`当前 / 总时长`，`timeupdate` 事件实时更新
- **进度条**：可点击拖拽 seek（按点击位置占比换算 `currentTime`）
- **关闭按钮**：暂停、清空 `src`、收起播放器、移除所有卡片高亮

### 4. 状态同步 `updateUI`

播放/暂停时同时更新**两处** UI：浮动播放器的主按钮，以及对应卡片的播放按钮和高亮态（`.music-card.playing` 加 accent 描边）。

## 样式要点

位于 [style.css:644-824](themes/Zebin-Theme/source/css/style.css#L644-L824)：

- **卡片播放按钮** `.music-play-btn`：默认 `display:none`，仅 `.has-audio` 显示；hover 卡片时也强制显示。
- **浮动播放器** `.music-player`：`position:fixed; bottom:0`，`.active` 控制显隐，`slideUp` 进场。
- **进度条** `.music-player-progress-wrap`：默认 3px，hover 撑高到 6px，bar 用 `--accent` 色。
- **深色模式**：`[data-theme="dark"]` 单独覆盖播放器背景与阴影。
- **响应式**：≤768px 隐藏时间显示、缩小按钮与内边距。

## 方案评价

**优点**

- 零运行时依赖，不引入 APlayer / MetingJS，加载快、维护简单。
- 音频托管在独立 GitHub 仓库，不占用博客仓库体积，CDN 直链。
- 卡片（跳转）与按钮（播放）职责分离，互不干扰（按钮 `stopPropagation`）。
- 单例 Audio + 复用，资源开销低。

**缺点 / 待改进**

- **硬编码映射表是最大债务**：新增/重命名歌曲要同时改 `music.yml` 和模板里的 `mf`，两处易脱节。理想的改进是把文件名也写进 `music.yml`（如增加 `audio` 字段），渲染时直接 `data-audio="<%- item.audio %>"`，彻底消除映射表。
- 文件名规范不统一（`_` 与 `-` 混用、空格有无不一），`encodeURIComponent` 虽能兜底，但增加了心智负担。
- 无播放列表 / 上一首下一首 / 音量控制；`ended` 仅复位 UI，不自动连播。
- raw.githubusercontent.com 在国内访问不稳定，可考虑 jsDelivr 镜像或自建对象存储。

## 重构方向（备忘）

若后续重构，最小改动路径：

1. `music.yml` 每条增加 `audio:` 字段（无则不写）。
2. 模板里渲染 `data-audio="<%- item.audio %>"`。
3. 删除 `mf` 映射表，`playSong` 改为直接读按钮的 `data-audio`。

即可把「硬编码」彻底改为「数据驱动」，与本站其他模块（movies / books）保持一致。
